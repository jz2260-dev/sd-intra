EMS Web App: Automatic Posting of Mediasite Content to Brightspace

Automatic posting of Mediasite recordings to Brightspace is requested in EMS by editing the class reservation details and setting the Brightspace posting option to Yes. An active recording request is required.

Support Workflow

Internal step path adapted from the source article.

  1. Confirm the class has an active recording request. The class must either remain opted in to recording or have a recording requested through AV Services for Classes.
  2. Have the user log in to the EMS Web App, or select Reserve a Room from the Stern Life portal.
  3. Have the user select My Events from the left navigation.
  4. If the user is submitting on behalf of someone else, have them select Act As from the menu under their name in the upper-right corner, then choose the appropriate user.
  5. If the target user does not appear, have the requester email ServiceDesk@stern.nyu.edu to request delegate access. Administrators may request access for faculty in their department. Teaching assistants should ask the instructor to send the request.
  6. From the reservations list, have the user select the class reservation that should post Mediasite content to Brightspace.
  7. Have the user open the Reservation Details tab.
  8. Have the user select Edit Reservation Details.
  9. Have the user scroll to the Additional Information section at the bottom of the page.
  10. Have the user change the Brightspace posting option to Yes.
  11. Have the user save or update the reservation details.
  12. To stop posting Mediasite content to Brightspace, have the user repeat the same steps and change the Brightspace posting option to No.
