Article Flow

Start with the first check and support workflow so the action path is visible immediately, then use triage and service facts for context.

First Check

Run these checks before spending time on quotas, refunds, or printer-specific troubleshooting.

1. Confirm eligibility User should have an active Stern ID or be a non-Stern student currently enrolled in a Stern course.
2. Confirm network If printing from a personal device, confirm the user is connected to the nyu Wi-Fi network.
3. Confirm release path Credits should only deduct after the user releases the job at a printer or release station.

Support Workflow

A tighter internal version of the troubleshooting flow.

  1. Validate eligibility. Confirm the user has an active Stern ID or is currently enrolled in a Stern course.
  2. Confirm network path. If the user is printing from a personal device, make sure they are on nyu Wi-Fi.
  3. Confirm queue target. Have the user send the job to the correct Print@Stern virtual printer.
  4. Confirm authentication. The user releases the job with their Stern ID, authenticated through the Stern Google account.
  5. Confirm release method. Direct the user to a release station or tap-to-release printer.
  6. Set expectations. Jobs expire after 24 hours and credits are deducted only after release.

Service Facts

High-confidence details to use after the initial checks and workflow are clear.

Eligible usersStern students with an active Stern ID and non-Stern students currently enrolled in a Stern course.
Not coveredFaculty, staff, administrators, teaching assistants, and other non-Stern students do not use Print@Stern.
Queue behaviorJobs stay in the user's Stern account queue for up to 24 hours, then delete automatically.
Credit deductionCredits are charged only when the user releases the job at a printer or release station.

Escalation Criteria

Escalate when the issue is account-specific, quota-related, or a valid release path fails.

Eligible user cannot access Print@Stern.

Quota appears incorrect or missing.

Credits were deducted for a job that did not print.

Tap-to-release does not recognize a valid Stern account.

User cannot purchase additional credits.

Club account requires account-specific assistance.
